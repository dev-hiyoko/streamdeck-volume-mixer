# Hiyoko Volume Mixer の使い方

Elgato Volume Controller が起動するローカル音声 WebSocket API
(`ws://127.0.0.1:1844`) を使って、**アプリごとの音量・ミュート**を
Stream Deck / Virtual Stream Deck のキーから操作するプラグインです。

「いま前面で鳴っているアプリだけ音量を下げる」「ゲームはそのままで BGM をミュート」
といった操作を、ミキサーを開かずワンタッチでこなせます。

## 前提

- Windows
- Stream Deck アプリ（7.1 以降）
- Elgato Marketplace の公式 `Volume Controller` プラグイン
  - これが裏で `ElgatoAudioControlServer.exe` を起動します。本プラグインはこのサーバー
    経由で音量を操作するため、**サーバーが動いていないとキーは効きません**。
  - サーバー未起動／起動前は、キーに **オフライン** と表示されます。起動後に自動復帰します。

## 設計方針

**固定プロファイルは生成しません。各キーが独立したアクション**として動作します。
物理 Stream Deck・Stream Deck Mobile・Virtual Stream Deck のいずれにも、
好きなレイアウトでキーを並べて使えます。

## アクションの置き方

カテゴリ「Hiyoko Volume Mixer」の **アプリ音量ミキサー** を、キーにドラッグして配置します。
設定画面は **このキーだけの設定** と **全キー共通（全体設定）** に分かれています。

## このキーの設定

- **操作**: `音量を上げる` / `音量を下げる` / `ミュート切替`
- **対象**:
  - `アプリ（自動スロット）`: 自動検出されたアプリのスロットを操作（既定）
  - `マスター（システム音量）`: システム既定の出力デバイス（マスター音量）を操作
- **スロット**: 操作するスロット位置（0 始まり。対象が「アプリ」のときのみ表示）

### スロットという考え方

このプラグインの肝は、**特定のアプリを名指ししない**点です。

- いま音を出しているアプリが、アクティブな順に自動で並びます。
- キーには「アプリ名」ではなく **スロット番号（0, 1, 2…）** を割り当てます。
  - スロット 0 = いちばんアクティブなアプリ
  - スロット 1 = その次 …
- つまり「スロット 0 のキー」は、**そのとき先頭に来たアプリを自動で掴む**キーになります。
  アプリを起動し直して PID が変わっても、キー側の設定を直す必要はありません。
- **空きスロットでもキーは配置できます。** そのスロットにアプリが来れば自動で効き始めます
  （アプリが来るまではキーに `—` が表示されます）。

### 検出中のアプリ（プレビュー）

対象が「アプリ」のとき、設定画面に **スロット → アプリ** の対応がリアルタイム表示されます
（このキーが選んでいるスロットを強調）。ここでは全キー共通の操作として、

- **表示名の変更**: 各アプリ名を上書き（空欄＝検出名のまま）。同じアプリが出る全キーに反映されます。
- **▲ / ▼ で優先順位の変更**: 上げたアプリほど小さいスロット番号（前の位置）に固定されます。

## 全体設定（全キー共通）

スロットの並び順や音量ステップは全キーで一致している必要があるため、これらは
**Stream Deck のグローバル設定**として保存され、すべてのキーで共有されます。
どれか 1 つのキーの設定画面で変えると、全キーに反映されます。

- **検出対象**: `アクティブのみ`（音が出ている／最近使われたアプリのみ）/ `すべて`
- **同名アプリをまとめる**: 同じ表示名のアプリを 1 グループにまとめる（既定 ON）
- **音量ステップ (%)**: 上げる／下げる 1 回あたりの増減幅（既定 5%）
- **更新間隔 (秒)**: 新しいアプリを検出するポーリング間隔（0.5〜10 秒、既定 1.5 秒）
  - 長くすると CPU 負荷は下がりますが、アプリの検出が遅くなります。

## ミキサーの組み方

1 アプリ（＝1 スロット）につき 3 キー（ミュート / 音量↑ / 音量↓）を、**同じスロット番号**で割り当てます。

| キー | 操作 | スロット |
| --- | --- | --- |
| 1 | ミュート切替 | 0 |
| 2 | 音量を上げる | 0 |
| 3 | 音量を下げる | 0 |
| 4 | ミュート切替 | 1 |
| 5 | 音量を上げる | 1 |
| 6 | 音量を下げる | 1 |

スロットを 0, 1, 2 … と増やすと、検出された別のアプリに割り当てられます。
アクティブなアプリが増減すると、その位置に来たアプリへ自動で切り替わります。
特定のアプリを常に同じ位置に置きたいときは、プレビューの ▲▼ で優先順位を固定してください。

## キーの見た目

キーは画像として自動描画されます（Stream Deck のタイトル文字は使いません）。

- **音量キー**: 上下のキーで **1 本のフェーダー**を分割表示します（上げる＝上半分、下げる＝下半分）。
  ノブが現在音量の位置に出るので、上げるキーの下に下げるキーを並べると、1 本の連続したフェーダー
  として読めます。`＋` / `－` のバッジと現在音量（%）も表示されます。
- **ミュートキー**: スピーカーアイコン。ミュート中は赤い斜線と「ミュート」、通常時は「通常」。
- **同名グループが複数**あるときは、名前の後ろに `×2` のように件数が付きます。
- **空きスロット**: アプリ名（または「スロット N」）と `—`。
- **オフライン**: 音声サーバーに繋がっていないとき。
- 「マスター」対象のキーは名前が「マスター」として描画されます。

## 自動検出と保存

- 対象は WebSocket から取得したアプリ一覧のうち、`アクティブのみ` では `activity <= 3`
  （再生中／直近使用）を優先して並べます。識別は表示名（`displayName`）です。
  システム全体（processID 0）はアプリ一覧から除外されます。
- **同名グループへの操作はグループ内の全アプリに一括適用**されます。
- 音量・ミュートは**出力デバイスごとに保存**され、アプリの再検出時に保存値へ自動同期されます。
  一度 Discord を 50% にしておけば、次に検出されたとき自動で 50% に戻ります。
- 音声サーバーの通知は取りこぼしがあるため、上記の更新間隔で定期的に検出も行います。

## 開発・インストール

```powershell
npm install
npm run build
.\scripts\install-dev-plugin.ps1
```

古いコードが残る場合は Stream Deck アプリを再起動してください。

## つまずきやすいポイント

- **キーが「オフライン」のまま** → `ElgatoAudioControlServer.exe` が動いていません。公式
  Volume Controller プラグインの導入と、サーバーの常駐を確認してください。
- **キーに `—`（空きスロット）が出る** → そのスロットに今アプリがいないだけです。対象アプリが
  音を出すと表示が戻ります。検出対象を「すべて」にすると無音アプリも並びに残せます。
- **狙ったアプリと違うものを操作してしまう** → スロットはアクティブ順に動的に並び替わります。
  位置を固定したいときはプレビューの ▲▼ で優先順位を設定してください。
